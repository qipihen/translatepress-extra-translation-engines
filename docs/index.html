What this plugin solves

  • TranslatePress OpenRouter support

  • TranslatePress Baidu translation and Youdao translation support
  • TranslatePress Tencent translation and Aliyun translation support

Designed for teams that need better cost, quality, and regional availability options.

Supported Translation Engines

Use these engines inside TranslatePress automatic translation settings.

OpenRouter / OpenAI Compatible

Route to OpenRouter or any OpenAI-compatible translation endpoint with configurable model and base URL.

Third-Party Proxy API

Single-token proxy mode for API gateway style translation providers. Example endpoint only, non-commercial reference.

Youdao Translate API

Official-style AppKey + AppSecret integration for Youdao translation workflows.

Baidu Translate API

Official-style AppID + AppSecret integration for Baidu translation workflows.

Tencent Cloud TMT

Tencent SecretId/SecretKey support with configurable region and optional session token.

Aliyun Machine Translation

Aliyun AccessKey support with region and scene options for production deployments.

Installation Guide

  1. Download the plugin package from the latest GitHub Release.
  2. Upload to WordPress via Plugins -> Add New -> Upload Plugin.
  3. Activate the plugin TranslatePress Extra Translation Engines.
  4. Configure API credentials in TranslatePress -> Automatic Translation.
  5. Test automatic translation with the built-in API credential test and start translating pages.

Developer and advanced settings references are available in repository docs (EN and ZH-CN).

FAQ

Does this plugin modify TranslatePress core files?

No. It extends TranslatePress behavior through hooks and compatible integration points.

Does it support OpenAI compatible translation interfaces?

Yes. You can use OpenRouter or other OpenAI-compatible API endpoints in the OpenRouter/OpenAI Compatible engine.

Is this plugin suitable for Chinese websites?

Yes. It supports domestic APIs such as Baidu, Youdao, Tencent Cloud TMT, and Aliyun Machine Translation.

Do I need separate API credentials for each provider?

Usually yes. Each official provider requires its own credentials. Proxy mode typically uses a single token.
